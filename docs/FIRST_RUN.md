# Первый запуск Cogwild

Документ описывает первую локальную проверку технического каркаса.

## Требования

- 64-битная Java 21;
- Git;
- IntelliJ IDEA или другая IDE с поддержкой Gradle;
- подключение к интернету для первого скачивания зависимостей.

## Подготовка

```bash
git clone https://github.com/Deenfoool/Cogwild.git
cd Cogwild
git checkout develop
```

На Windows все команды ниже выполняются из корня репозитория.

## Проверка Java

```bat
java -version
```

Ожидается Java 21. Если отображается другая версия, настройте `JAVA_HOME` на JDK 21.

## Проверка Wrapper

```bat
gradlew.bat --version
```

Команда должна вывести сведения о Gradle и JVM без ошибок.

## Первая сборка

```bat
gradlew.bat build --stacktrace
```

Готовый JAR после успешной сборки будет находиться в:

```text
build/libs/
```

## Запуск клиента

```bat
gradlew.bat runClient
```

Первый запуск может занять несколько минут из-за загрузки зависимостей и подготовки окружения.

## Проверка в игре

1. Убедиться, что Cogwild присутствует в списке модов.
2. Создать творческий мир.
3. Выполнить:

```mcfunction
/give @s cogwild:steel_ingot
/give @s cogwild:machine_casing
```

4. Проверить отображение предмета.
5. Поставить и сломать корпус машины.
6. Убедиться, что блок выпадает после разрушения подходящим инструментом.
7. Проверить журнал на ошибки регистрации и GeckoLib.

## Что приложить к сообщению об ошибке

- полный текст ошибки, начиная с `FAILURE: Build failed with an exception`;
- файл `logs/latest.log`, если клиент запустился;
- версию Java из `java -version`;
- команду, после которой возникла ошибка;
- номер коммита или название ветки.

## Критерий успешного первого запуска

- `gradlew.bat build` завершился сообщением `BUILD SUCCESSFUL`;
- `gradlew.bat runClient` открыл Minecraft;
- оба объекта выдаются командами;
- корпус машины размещается и ломается;
- критических ошибок в журнале нет.
